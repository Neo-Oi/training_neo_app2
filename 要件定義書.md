# 要件定義書

## 1. 概要

### 1.1 システム化の目的

研修で、ログイン機能、権限制御、ユーザー管理機能の設計・実装・試験を行うため。

### 1.2 期限

ログイン機能およびユーザー管理機能について、設計から試験、報告書作成までを10営業日で行う。

## 2. 利用者要件

| 利用者区分 | 説明 |
|---|---|
| 一般ユーザー | ログイン・ログアウトができるユーザー。管理機能は利用できない。 |
| 管理者ユーザー | ログイン・ログアウトに加えて、ユーザー管理機能を利用できるユーザー。 |
| 未ログインユーザー | ログイン前のユーザー。認証が必要な画面にはアクセスできない。 |

## 3. 機能要件

- ログインできること
- ログアウトできること
- パスワードが安全に保存されること
- ログイン状態の管理ができること
- ログインの失敗回数が制御できること
- 未ログイン時のアクセスが制御できること
- 一般ユーザーと管理者ユーザーの権限区分があること
- 管理者ユーザーがユーザーの新規作成、閲覧、編集、削除をできること
- ユーザー作成時にメールアドレスが重複しないこと
- ユーザー作成時のパスワードには文字数などのバリデーションを設けること

## 4. 非機能要件

- パスワードをそのまま保存しないこと
- 未ログインユーザーはログイン済みユーザー向け画面に遷移できないこと
- 一般ユーザーは管理者ユーザー向け画面に遷移できないこと
- ログイン、ログアウトが画面上で行えること
- ログイン機能、ユーザー管理機能、権限区分の役割を分けて実装すること
- 入力エラー時には登録、更新を行わないこと

## 5. 対象外機能

- メール認証
- パスワード再設定
- 外部サービス認証
- 画像アップロード
- 詳細な操作ログ管理
- member/admin以外の複雑な権限ロール
- 本番公開

## 6. 画面要件

| 画面名 | 利用者 | 目的 |
|---|---|---|
| ログイン画面 | 未ログインユーザー | ログイン用 |
| ユーザー詳細画面 | 一般ユーザー、管理者ユーザー | ユーザー情報確認用 |
| ユーザー一覧画面 | 管理者ユーザー | 作成されているユーザー一覧確認用 |
| ユーザー新規作成画面 | 管理者ユーザー | ユーザーの新規作成用 |
| ユーザー情報編集画面 | 管理者ユーザー | ユーザー情報および権限の編集用 |
