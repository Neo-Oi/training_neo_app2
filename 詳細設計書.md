# 詳細設計書

## 1. 概要

本書では、ユーザー管理システムの詳細設計を定義する。  
対象は、routes設計、Controller設計、Model設計、処理フロー、シーケンス、バリデーション、seed.rb、画面マークアップ、テスト観点とする。

## 2. routes設計

| URL | HTTPメソッド | Controller#Action | 用途 |
|---|---|---|---|
| `/users/sign_in` | GET | Devise | ログイン画面表示 |
| `/users/sign_in` | POST | Devise | ログイン処理 |
| `/users/sign_out` | DELETE | Devise | ログアウト処理 |
| `/users` | GET | users#index | ユーザー一覧表示 |
| `/users/:id` | GET | users#show | ユーザー詳細表示 |
| `/users/new` | GET | users#new | ユーザー新規作成画面表示 |
| `/users` | POST | users#create | ユーザー作成処理 |
| `/users/:id/edit` | GET | users#edit | ユーザー編集画面表示 |
| `/users/:id` | PATCH | users#update | ユーザー更新処理 |
| `/users/:id` | DELETE | users#destroy | ユーザー削除処理 |

## 3. Controller設計

### ApplicationController

| 処理 | 内容 |
|---|---|
| ログイン後遷移 | ログイン成功後、現在ログイン中ユーザー自身の詳細画面へ遷移する |
| ログアウト後遷移 | ログイン画面へ遷移する |

ログイン後は、一般ユーザー・管理者ユーザーに関係なく、自身のユーザー詳細画面へ遷移する。

### UsersController

| アクション | 内容 | 利用者 |
|---|---|---|
| index | ユーザー一覧を表示する | 管理者ユーザー |
| show | ユーザー詳細を表示する | 一般ユーザー / 管理者ユーザー |
| new | ユーザー新規作成画面を表示する | 管理者ユーザー |
| create | ユーザーを新規作成する | 管理者ユーザー |
| edit | ユーザー編集画面を表示する | 管理者ユーザー |
| update | ユーザー情報を更新する | 管理者ユーザー |
| destroy | ユーザーを削除する | 管理者ユーザー |

### before_action

| 処理 | 内容 |
|---|---|
| user_init | 未ログインユーザーをログイン画面へ遷移させる |
| user_role_init | 管理者ユーザー以外が管理者用画面へアクセスできないようにする |

## 4. Model設計

### Userモデル

| 項目 | 内容 |
|---|---|
| 認証 | Deviseを使用する |
| 権限区分 | roleでmember/adminを判定する |
| ログイン失敗制御 | Devise Lockableを使用する |
| パスワード管理 | Deviseによりハッシュ化して保存する |

roleは以下の2種類とする。

| role | 内容 |
|---|---|
| member | 一般ユーザー |
| admin | 管理者ユーザー |

## 5. 処理フロー

### ログイン処理

1. ユーザーがログイン画面でメールアドレスとパスワードを入力する。
2. 入力されたメールアドレスのユーザーが存在するか確認する。
3. ユーザーが存在する場合、入力されたパスワードが一致するか確認する。
4. メールアドレスまたはパスワードが一致しない場合、エラーメッセージを表示し、再入力させる。
5. ログインに3回失敗した場合、アカウントを15秒間ロックする。
6. 認証に成功した場合、ログイン中ユーザー自身の詳細画面へ遷移する。

### ユーザー詳細表示処理

1. ユーザー詳細画面へアクセスする。
2. `user_init` によりログイン状態を確認する。
3. 未ログインの場合はログイン画面へリダイレクトする。
4. ログイン済みの場合、ユーザー詳細画面を表示する。
5. 画面上にログアウトボタンを表示する。
6. 管理者ユーザーの場合、管理者画面ボタンを表示する。

### ログアウト処理

1. ユーザーがログアウトボタンを押す。
2. Deviseによりログアウト処理を行う。
3. 現在ログイン中ユーザーのセッションを破棄する。
4. ログイン画面へ遷移する。

ログアウト対象は、画面に表示されているユーザーではなく、現在ログインしているユーザーとする。

### 管理者画面遷移処理

1. ユーザー詳細画面で管理者画面ボタンを押す。
2. ユーザー一覧画面へ遷移しようとする。
3. `user_role_init` によりroleを確認する。
4. 一般ユーザーの場合、自身の詳細画面へリダイレクトする。
5. 管理者ユーザーの場合、ユーザー一覧画面を表示する。

### ユーザー一覧表示処理

1. ユーザー一覧画面へアクセスする。
2. `user_init` によりログイン状態を確認する。
3. 未ログインの場合はログイン画面へリダイレクトする。
4. ログイン済みの場合、`user_role_init` によりroleを確認する。
5. 一般ユーザーの場合、自身の詳細画面へリダイレクトする。
6. 管理者ユーザーの場合、ユーザー一覧画面を表示する。

### ユーザー新規作成処理

1. 管理者ユーザーがユーザー新規作成画面へアクセスする。
2. `user_init` によりログイン状態を確認する。
3. `user_role_init` によりroleを確認する。
4. 管理者ユーザーの場合、新規作成画面を表示する。
5. 名前、メールアドレス、パスワード、権限を入力する。
6. 作成処理時にバリデーションを行う。
7. 問題がなければユーザーを作成する。
8. 作成後、ユーザー一覧画面へ遷移する。
9. 入力エラーがある場合は登録せず、新規作成画面を再表示する。

### ユーザー編集処理

1. 管理者ユーザーがユーザー編集画面へアクセスする。
2. `user_init` によりログイン状態を確認する。
3. `user_role_init` によりroleを確認する。
4. 管理者ユーザーの場合、`params[:id]` から対象ユーザーを取得する。
5. 対象ユーザーの編集画面を表示する。
6. 入力内容を変更する。
7. 更新処理時にバリデーションを行う。
8. 問題がなければユーザー情報を更新する。
9. 更新後、ユーザー一覧画面へ遷移する。
10. 入力エラーがある場合は更新せず、編集画面を再表示する。

### ユーザー削除処理

1. 管理者ユーザーが削除ボタンを押す。
2. `user_init` によりログイン状態を確認する。
3. `user_role_init` によりroleを確認する。
4. 管理者ユーザーの場合、`params[:id]` から対象ユーザーを取得する。
5. 対象ユーザーを削除する。
6. 削除後、ユーザー一覧画面へ遷移する。

## 6. シーケンス設計

### ログイン処理

| 順番 | 処理 |
|---|---|
| 1 | ユーザーがログイン画面でメールアドレスとパスワードを入力する |
| 2 | ログイン画面からDeviseへログイン要求を送る |
| 3 | DeviseがUserモデルに認証処理を依頼する |
| 4 | Userモデルがusersテーブルからメールアドレスに一致するユーザーを検索する |
| 5 | ユーザーが存在する場合、入力されたパスワードと保存済みパスワードを照合する |
| 6 | 認証に成功した場合、現在ログイン中ユーザー自身の詳細画面へ遷移する |
| 7 | 認証に失敗した場合、エラーメッセージを表示する |
| 8 | ログイン失敗が3回に達した場合、アカウントを15秒間ロックする |

### 管理者画面遷移処理

| 順番 | 処理 |
|---|---|
| 1 | ユーザー詳細画面で管理者画面ボタンを押す |
| 2 | users#indexへアクセスする |
| 3 | UsersControllerでログイン状態を確認する |
| 4 | 未ログインの場合、ログイン画面へリダイレクトする |
| 5 | ログイン済みの場合、user_role_initでroleを確認する |
| 6 | 一般ユーザーの場合、自身の詳細画面へリダイレクトする |
| 7 | 管理者ユーザーの場合、usersテーブルからユーザー一覧を取得する |
| 8 | ユーザー一覧画面を表示する |

### ユーザー作成処理

| 順番 | 処理 |
|---|---|
| 1 | 管理者ユーザーがユーザー新規作成画面を開く |
| 2 | 名前、メールアドレス、パスワード、権限を入力する |
| 3 | 登録ボタンを押す |
| 4 | UsersControllerのcreateアクションで入力値を受け取る |
| 5 | Userモデルでバリデーションを行う |
| 6 | バリデーションに問題がない場合、usersテーブルにユーザーを登録する |
| 7 | 登録後、ユーザー一覧画面へ遷移する |
| 8 | バリデーションエラーがある場合、登録せず新規作成画面を再表示する |

### ユーザー更新処理

| 順番 | 処理 |
|---|---|
| 1 | 管理者ユーザーがユーザー編集画面を開く |
| 2 | UsersControllerのeditアクションでparams[:id]から対象ユーザーを取得する |
| 3 | 編集画面に対象ユーザーの情報を表示する |
| 4 | 管理者ユーザーが入力内容を変更する |
| 5 | 更新ボタンを押す |
| 6 | UsersControllerのupdateアクションで入力値を受け取る |
| 7 | Userモデルでバリデーションを行う |
| 8 | バリデーションに問題がない場合、usersテーブルのユーザー情報を更新する |
| 9 | 更新後、ユーザー一覧画面へ遷移する |
| 10 | バリデーションエラーがある場合、更新せず編集画面を再表示する |

### ユーザー削除処理

| 順番 | 処理 |
|---|---|
| 1 | 管理者ユーザーがユーザー一覧画面で削除ボタンを押す |
| 2 | UsersControllerのdestroyアクションでparams[:id]から対象ユーザーを取得する |
| 3 | 対象ユーザーをusersテーブルから削除する |
| 4 | 削除後、ユーザー一覧画面へ遷移する |

## 7. バリデーション設計

| 項目 | バリデーション |
|---|---|
| name | 必須 |
| email | 必須、重複不可、メール形式(hoge@fuga.hoge) |
| password | 必須、6文字以上 |
| role | memberまたはadminのいずれか |

ユーザー作成・更新時には、入力値に対してバリデーションを行う。  
バリデーションエラーがある場合は、登録・更新を行わず、入力画面を再表示する。

## 8. seed.rb設計

開発・確認用として、以下のユーザーをseed.rbで作成する。

| 種別 | name | email | password | role |
|---|---|---|---|---|
| 一般ユーザー | abc | abc@example.com | password | member |
| 一般ユーザー | def | def@example.com | password | member |
| 一般ユーザー | xyz | xyz@example.com | password | member |
| 管理者ユーザー | admin | admin@example.com | password | admin |

パスワードはDeviseのpassword属性を使用して設定する。  
encrypted_passwordには直接値を設定しない。
